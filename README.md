# nearest_bars

Проект предназначен для отображения на html-карте ближайших 5 (пяти) баров/питейных заведений.
Для работы требуется база данных баров Москвы (`bars_db.json`).
На вход подается адрес, ближайшие бары к которому будут подписанными маркерами отмечены на карте.

## Требования

- Python 3.10+
- API токен от [LocationIQ](https://locationiq.com/)

### Установка и запуск с Docker

```bash
docker-compose up --build
```

Сервис будет доступен на порту 8001.

### Установка и запуск без Docker

1. Скачайте файлы проекта
2. Скачайте базу данных баров Москвы и положите файл `bars_db.json` в папку проекта
3. Создайте файл `.env` с вашим API токеном:

```
API_TOKEN=ваш_токен
```

4. Установите зависимости:

```bash
python -m venv venv
venv\Scripts\activate  # Windows
source venv/bin/activate  # Linux/Mac
pip install -r requirements.txt
```

5. Запустите программу:

```bash
python main.py "Москва Тучковская улица 3"
```

### Тестовый режим

Для отладки используется временный файл .json. По умолчанию он очищается.
Для использования кэша добавьте флаг `--test`:

```bash
python main.py "Москва Тучковская улица 3" --test
```

### Линтинг и проверка кода

| Переменная | Описание |
|------------|----------|
| `API_TOKEN` | API токен от LocationIQ |

### Обработка сетевых ошибок и тайм-ауты

- Запросы к LocationIQ выполняются с тайм-аутом 10 секунд и с двумя повторными попытками при временных ошибках сети.
- При невозможности получить координаты функция возвращает None и логирует ошибку.
- При отсутствии валидных координат для баров приложение продолжает работу, пропуская проблемные записи.
- Все сетевые ошибки логируются для упрощения диагностики.
